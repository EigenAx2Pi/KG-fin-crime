The silver layer knowledge graph uses four entity types and five relationship types aligned to the FIBO financial ontology. Every transaction in the graph is represented as a FINANCIAL_TRANSACTION node connected to source and destination ACCOUNT nodes, each owned by a PARTY and held at a FINANCIAL_INSTITUTION.
